# Appurdex Working Rules

- Do not invent mock/sample/placeholder tool rows, metrics, API responses, or
  benchmark scores.
- If a real endpoint, repository, or benchmark source is missing, show an honest
  empty/error/unknown state.
- Keep the first screen as the live directory/table view, not a marketing hero.
- Treat freshness and last-verified indicators as core product UI.
- Avoid web scraping in the MVP unless the user explicitly changes scope.
- Route visible "Website" links to the AI product's main website first. Use a
  GitHub repository URL only as a last resort, and when unsure, get the
  website from the repository homepage metadata instead of treating the repo as
  the website.
